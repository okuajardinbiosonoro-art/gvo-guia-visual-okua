# ADR — Architecture Decision Records

Aquí se registran decisiones estructurales del proyecto.

## Cuándo crear un ADR

Crear un ADR cuando se tome una decisión que:
- afecte varios módulos;
- cambie stack o herramientas principales;
- condicione despliegue u operación;
- tenga trade-offs relevantes;
- sea difícil de recordar tiempo después.

Usar la plantilla `ADR-000-template.md`.
