# Skill: vertical-slice

Usa esta skill para convertir una necesidad del producto en un incremento pequeño, cerrable y comprobable.

## Objetivo

Definir o ejecutar un cambio que cruce varias capas sin crecer demasiado:
- docs;
- contrato;
- frontend;
- backend;
- prueba mínima.

## Reglas

1. Empezar por el valor al usuario o al sistema.
2. Limitar el alcance a un incremento pequeño.
3. No mezclar refactor grande con feature grande.
4. Dejar claro qué queda fuera.
5. Incluir validación automática o manual mínima.

## Plantilla de trabajo

- problema
- objetivo del slice
- archivos a tocar
- orden de implementación
- validaciones
- riesgos
- documentación a actualizar
